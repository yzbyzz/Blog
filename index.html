C++编程范型（Programming Paradigms）

  • 基于过程 Procedural-based
  • 基于对象 Object-based
  • 面向对象 Object-oriented
  • 泛型技术 Generics
    C++断想

中英简繁术语对照

  • object 在Object-oriented中的真正意义是“物体、物件”，而非“对象、目标”

0 导读

语（Terminology）

  • 声明式
    告诉编译器某个东西的名称和类型，但略去细节。
  • size_t是一个typedef，unsigned类型。
  • 函数的声明揭示其签名式，也就是参数和返回类型。一个函数的签名等同于该函数的类型。
  • C++对签名式的官方定义并不包括函数的返回类型。

  • 定义式
    提供编译器一些声明式所遗漏的细节。

  • 初始化
    “给予对象初值”的过程。

    • 构造函数声明为explicit，阻止构造函数被用来执行隐式类型转换，但仍可被用来执行显示类型转换。
  • copy构造函数
    以同类型对象初始化自我对象
  • copy assignment操作符
    从另一个同型对象拷贝其值到自我对象

  • Pass-by-value
    调用copy构造函数

  • STL（标准模板库）
    许多相关机能以函数对象（function object，“行为像函数”的对象，重载operator( )（function call操作符））实现。

  • 不明确行为（undefined behavior）
    尽可能避免

  • 接口
    C++没有像Java、.Net语言那样的接口
    一般指函数的签名或class的可访问元素或针对某template类型参数需为有效的一个表达式

  • 客户
    使用你所写代码（通常是一些接口）的人或物。
    函数的客户：其使用者，程序中调用函数（或取其地址）的那部分，编写并维护那些代码的人。
    书中常指程序员。

TR1（“Technical Report 1”）

  • 一份规范，描述加入C++标准程序库的诸多新机能
  • 这些机能以新的class templates和function templates形式体现
  • 针对的题目有hash table，reference-counting smart pointers，regular expressions，以及更多
  • 所有TR1组件都被置于命名空间tr1内，后者嵌套于命名空间std内

    Boost

  • 组织，亦是网站，提供可移植、同僚复审、源码开放的C++程序库。
  • 大多数TR1机能是以Boost的工作为基础。
  • Boost提供比TR1更多的东西
